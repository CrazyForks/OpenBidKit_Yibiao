# 易标用户手册

基于项目代码和发行版真实界面制作手册。以用户可见的发行版为最终依据，不根据未发布代码臆造功能。

## 开始前

1. 完整阅读 [project-contract.md](references/project-contract.md)。
2. 从当前工作目录向上查找 `AGENTS.md` 并遵守；进入客户端代码前阅读 `client/开发说明.md`。
3. 检查工作区状态，保留用户已有修改，不修改 `client/`、`analytics/` 或范围外手册内容。
4. 解析用户请求并建立目标清单：目标 Markdown、原图、标注图。用户指定一个或多个功能时，只把这些文件列入目标；未指定时才使用完整手册范围。
5. 对范围外手册文件记录修改前哈希；完成后复核哈希未变化。

## 确认发行版

按以下顺序确认可执行文件：

1. 使用用户本次明确提供的有效路径。
2. 否则使用项目参考中的默认路径，但必须确认文件存在且产品名称为“易标投标工具箱”。
3. 默认路径失效、路径未知或出现多个候选时，立即询问用户提供发行版路径并暂停；不得运行 Vite/Electron 开发版代替。

需要控制 Windows 界面时，使用可用的 Windows 应用控制能力并遵守其说明。打开软件后只导航、展开、切换页签或点击“下一步”查看已有结果，不点击重新生成、开始解析、模型测试、智能体自检、重新检查或导出等会执行任务的按钮。

## 检查数据门槛

进入目标功能后先判断截图所需的数据是否存在：

- 配置页只需界面可见，不要求业务结果。
- 流程页必须已经有导入文件、解析结果、目录、正文、知识库文档或检查结果等与目标章节匹配的数据。
- 只有空白页、零条记录、占位内容或未执行状态时，视为没有数据。

没有数据时停止文档制作，不创建推测性说明，也不自行运行流程。明确告诉用户缺少哪个功能的数据、需要在发行版中完成哪些步骤，并提示完成后再次调用本 Skill。若用户限定了范围，只检查该范围的数据门槛。

## 创建或更新文档

1. 对照 `menuConfig.ts`、`AppRouter.tsx`、目标 `features/` 页面和发行版界面，确认功能已完成且普通用户可稳定使用。
2. 跳过“正在开发中”、禁用、开发者模式专用或只存在于代码而发行版不可见的功能。
3. 手册不存在时创建标准目录；手册存在时保留未变化章节和截图。
4. 使用简单中文说明“在哪里点、等待什么、如何确认完成、最后得到什么”。界面按钮和菜单名称保持原文。
5. 用户只要求编写或更新某部分时，只编辑目标 Markdown 和目标截图；范围外问题仅报告，不修复。
6. 不创建根 README、总目录、上一篇、下一篇、返回链接或其他章节导航。

## 采集和标注截图

1. 最大化发行版窗口，等待动画和提示消失，避免鼠标遮挡关键内容。
2. 使用 [capture-window.ps1](scripts/capture-window.ps1) 按物理像素窗口边界保存无损 PNG 到 `使用说明/images/`。
3. 目视检查原图清晰、页面状态正确、API Key 为掩码且没有不应公开的信息。
4. 阅读 [annotation-format.md](references/annotation-format.md)，为每张图准备 1–5 个短标注的 JSON 坐标。
5. 使用 [annotate-screenshot.ps1](scripts/annotate-screenshot.ps1) 生成 `使用说明/images/标注/` 下的同名 PNG。最终标注必须由该确定性脚本叠加，不使用生成式图片编辑重绘界面。
6. 目视检查标签不遮挡关键文字、箭头准确落到按钮或需复核区域，且标注图尺寸与原图一致。

## 验证和交付

1. 使用 [validate-manual.ps1](scripts/validate-manual.ps1) 校验目标范围；未限定范围时校验完整手册。
2. 限定范围时仍以只读方式扫描完整手册，范围外断链只报告，不改动。
3. 复核范围外文件哈希、工作区差异和目标图片引用，确认没有越界修改。
4. 如果本次由 Skill 启动软件，完成后关闭；如果软件原本已由用户打开，则保持打开并说明当前停留页面。
5. 最终只汇报处理范围、创建或更新的章节、截图数量、验证结果和需要用户补充的数据。

## 常用调用

- `$yibiao-user-manual`：创建或更新完整手册。
- `$yibiao-user-manual 只更新废标项检查文档`：只处理指定功能。
- `$yibiao-user-manual 重新采集标书查重截图`：只替换该功能截图。
- `$yibiao-user-manual 仅检查，不修改`：执行只读检查。
